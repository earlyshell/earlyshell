Humanoid Robot 2011
Description

Robots, eh? This product was made for educational purposes - a thesis, back in college, which demonstrates application of concepts relating to robotics such as kinematics, certain mathematical models, COG (weight management), etc.

This robot was created, generally, to market and boost interest for robotics in the country (PH). Standing 17" in height, with 20-DOF (Degrees of Freedom), humanoid in form, the team has aimed to make such system, flexible enough to perform several anthropomorphic functions. More specifically, turning, balancing with one leg, walking up (and down, ambitiously) the stairs, crouch, punch, kick, dance, and identify obstacles. But, of course, such has certain limitations as the scope is dependent on the time the team had during college.

Although this may all sound promising but, the team really had a hard time (difficult times) due to the complexity and unpreparedness for such endeavor, and was only able to deliver minimally. It also took 4 re-designs due to unforeseen issues. Not proud of this, but sometime in the future, I will initiate a remake of this project. Now, that's exciting news! For now, let me present to you some details and pictures of the project.

Details/Specifications
Proximity Sensor
  • Limited to 1 sensor only (just to detect frontal obstacle)
  • Stops at any command/motion when obstacle is detected
  • IR Proximity Sensor Kit
Gyroscope
  • Orientation Feedback
  • Not implemented
Motor Controllers
  • 2 units: 1 - upper limbs/torso; 1 - lower limbs
  • e-Gizmo 8/16 Channel Servo Controller - a universal servo controller that can control individual movements of up to 16 servo motors in a single board
Body / Frame
  • Polycarbonate (PC1000)
  • Lightweight
  • Good machinability (Fabrication)
  • Durable (doesn't easily break)
Motor / Actuators (Upper-limbs/Torso)
  • Tower Pro MG995 Servo Motor
  • Input Voltage: 3.5 to 8.4V
  • Speed (4.8V): 0.17 sec/60°
  • Speed (6.0V): 0.13 sec/60°
  • Torque (4.8V): 180 oz-in (13kg-cm)
  • Torque (6.0V): 208 oz-in (15kg-cm)
  • Dimensions: 1.57" x 0.79" x 1.44" (40mm x 20mm x 36.5mm)
  • Weight: 1.78oz (48g)
Motor / Actuators (Lower-limbs)
  • Futaba S9351 Servo Motor
  • High-torque, Coreless Motor, Metal Gears, High-current Capable Wiring
  • Speed (4.8V): 0.16 sec/60°
  • Speed (6.0V): 0.13 sec/60°
  • Torque (4.8V): 153 oz-in
  • Torque (6.0V): 192 oz-in
  • Dimensions: 1.6" x 0.8" x 1.4" (40mm x 20mm x 37mm)
  • Weight: 2.2oz (61g)
Power Supply
  • Industrial Power Supply (e-Gizmo)
  • Input Voltage: 100-125VAC
  • Output Voltage: 5VDC 20A
Microcontroller
  • IRC Slimboard - Zilog (e-Gizmo)
  • Z8F6421 Flash Microcontroller - a Flash-capable microcontroller based on Zilog's advanced eZ8-8bit microprocessor core
  • Input: 7VDC
Motion Planning / Simulation
  • MATLAB - Robotics Toolbox (by Peter Corke)
Images
Summary/Take-away(s)
  • Top-heavy (not ideal for humanoid robots)
  • Power concerns (under-performing power sources)
  • Motor positioning cannot accomodate perfect motions (causing improper distribution of weight)
  • Revisit mathematical models and re-create algorithms
  • Expected movements were performed but barely satisfactory
  • For re-make in the near future. Will use Arduino or a more powerful microcontroller kit that will be available in the future. ;)

Line-tracing Robot 2009
Description

Part of the cool things that we do, back in college, is this - a line-tracing robot. This project is basically an application of our lessons in microcontrollers. For this example, we used an 8051 dev-board - fabricated using a template (printed, etched, soldered), coded, compiled, built, loaded instructions to the microcontroller, and go!

Details/Specifications
Body / Frame
  • Recycled Illustration Board
  • Ferrero Rocher Container
  • Other recycled materials
Motors / Actuators / Drivers
  • (2) Hyper Dash 2
  • Gear-train (supporting common DC motors from toys) - to increase torque and support (or act as alternative for) the H-bridge's low current output and carry the load (weight).
  • H-bridge - motor driver
Sensors / Environment
  • (3) Phototransistors
  • Each Phototransistor is partnered with a super-bright LED
  • Test environment is controlled; an illustration board wrapped with tapes, reflective enough to signal the phototransistor; path is built using electical tapes (non-reflective)
  • Two of the sensors are used to determine direction (left / right)
  • Third sensor is to determine whether the robot is still on the path
Microcontroller
  • 8051 Dev-Board (Template)
  • Used Kiel uVision (C/C++) to code instructions
  • Used 8051 Loader to burn instructions to the microcontroller
  • High-level Instruction: If reflecting light, proceed. If non-reflective is detected (by left or right sensor), turn accordingly. Third sensor is to detect if the robot is still on the path. No instructions provided for conflicting/branched path.

4 x 5 - LED Display 2009
Description

Yet another microcontroller application. This time, I created a 4 x 5 LED display to apply/show a 7-segment display. The 4 x 5 display is an add-on, sort of an actuator for the microcontroller.

Details/Specifications
Microcontroller
  • 8051 Dev-Board (Template)
  • Used Kiel uVision (C/C++) to code instructions
  • Used 8051 Loader to burn instructions to the microcontroller
  • High-level Instruction: Display 0-9 as inputted in HyperTerminal
4 x 5 Display
  • 20 LEDs - Orange (Regular)
  • 8-bit Input
  • Board fabricated by Team Go Large
